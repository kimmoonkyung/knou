## 요약3_멀티미디어 데이터
```
  01. 텍스트(text)는 숫자와 문자로 구성된 문서 데이터로 종이 위에 인쇄되거나 모니터 등의 화면상에 표시되는 정보이다. 다른 멀티미디어 데이터처럼 시청각적인 흥미를 제공하지는 않지만
      중요하고 정교한 정보를 옮기는 경우에 사용된다.
  
  02. 문자집합을 코드 형태로 표기한 것을 코드화 된 문자집합(CCS, Coded Character Set)이라 하며 텍스트는 문자의 집합으로 구성된다. 모든 문자는 컴퓨터 내부에서 2진수의 코드료 표시되고,
      각각의 문자는 1개의 숫자로 할당된다. 이것을 문자 인코딩이라 한다.
  
  03. ASCII(American Standard Code for Information Interchange)는 미국에서 제정한 표준으로 컴퓨터나 인터넷상에서 텍스트 파일을 만드는 데 사용되는 일반적인 형식으로
      총 129개의 문자가 정의되어 있고, EBCDIC(Extended Binary Coded Decimal for Interchange Code)는 IBM사의 중형 컴퓨터시스템에 사용하기 위해 가발된 코드 체계로
      현재는 ASCII가 주로 사용됨에 따라 잘 사용되지 않는다.
  
  04. 한글코드는 조합형과 완성형 코드가 있다. 조합형 한글코드는 초성, 중성, 종성에 각각 코드를 할당하는 방식이고, 완성형 한글코드는 '가', '각', '간'과 같은 완성 된 문자에 코드를 할당하는 방식
  
  05. 유니코드는 전 세계의 모든 문자를 컴퓨터에서 일관되게 표현하고 다룰 수 있도록 설계 된 산업표준으로 유니코드 협회가 제정한다.
  
  06. 마크업 텍스트란 문서의 논리 구조나 체제와 같은 문서의 서식 작성을 지정하거나, 문서 내용의 찾아보기, 단어나 구, 화상과 같은 요소를 같은 문서 또는 다른 문서 중의 다른 요소와 연결하는 방법을
      지시하는 마크업 정보가 추가 된 것이다.
  
  07. 하이퍼텍스트란 비순차적인 텍스트로서 여러 개의 연결 된 문서라는 의미이다. 하이퍼텍스트 문서는 물리적이고 논리적인 공간을 뛰어넘는 노드와 링크라는 임의의 연결이 가능한 새로운 개념의 문서 연결 형태이다.
  
  08. TXT는 TXT 포맷으로 운영체제에서 기본적으로 제공하는 텍스트 편집기를 사용하여 만들어진 파일 형식이다.
  
  09. DOC는 마이크로소프트사의 MS워드를 사용하여 작성 된 문서 포맷이다. 그림 및 특수기호, 서식 정보, 탭, 줄간격 등을 포함하며 확장자는 "*.docx"이다.
  
  10. HWP는 한글 워드프로세서에서 사용하는 문서 포맷이다. 다른 워드프로세서의 경우 완성형 코드를 사용하는 데 비해 조합형 코드를 사용하고 있는 특징이 있다.
  
  11. RTF(Rich Text Format)는 미국의 마이크로소프트사가 중심이 되어 표준화한 텍스트 문서 파일 형식으로 서로 다른 운영체제와 워드프로세서 사이에서도 텍스트 파일을 교환 할 수 있도록 하기 위해서 사용 된 
      문서 형식이다.
  
  12. HTML(Hyper Text Markup Language)문서 형식은 일반 TXT 형식으로 되어 있으므로 일반 TXT를 편집할 수 있는 문서 편집기라면 조작이 가능하다.
  
  13. XML(eXtensible Markup Language)은 1996년에 하이퍼텍스트 생성 언어인 HTML을 대체할 목적으로 월드 와이드 웹 컨소시엄에서 제안하고 표준화 작업을 진행한 페이지 기술 언어이다.
  
  14. PDF(Potable Document Format)는 어도비 시스템즈에서 개발한 전자문서 형식이다. PDF는 컴퓨터 환경에 관계없이 같은 표현을 하기 위한 목적으로 개발되었다.
  
  15. 테크(TeX)는 상세 지정이 가능한 문서 정형 또는 조판 시스템으로 유닉스상에서 개발된 일괄(batch) 방식의 형식이고, 라테크(LaTeX)는 테크가 갖고 있는 사용자 인터페이스의 불편함으로 인해 일반인들이
      쓰기 어려운 점을 개선 한 것이다.
      
  16. 전자도서는 전자책, 흔히 e- Book(Electronic Book)이라고 불리며, 온라인 전자출판의 일종이다. 즉, 책 내용을 디지털 파일로 만들어 전용 뷰어를 통해 컴퓨터나 전용 단말기로 읽는다.
  
  17. 전자출판(ePublishing)이란 컴퓨터를 이용하여 디지털화 된 방식으로 출판물을 제작하는 방법과 기술을 의미한다.
```

```
    1) ASCII            - 정답
    2) EBCDIC
    3) 서유럽 문자집합
    4) 한글코드
  
  2. 마크업 텍스트의 설명으로 틀린 것은 ?
    1) 문서의 논리 구조나 체제와 같은 문서의 서식 작성을 지정한다.
    2) 문서 내용의 찾아보기 또는 찾아보기 작업 방법을 지정한다.
    3) 구조적인 경계들을 밝히는 특수한 문자를 삽입하는 표현법이다.
    4) 문서 또는 다른 문서 중의 다른 요소와 연결하는 방법을 지정하지 못한다.     - 정답
    
  3. HTML에 대한 설명으로 옳지 않는 것은 ?
    1) 문서의 형식은 TXT 형식으로 되어 있다.
    2) 단순한 텍스트 파일 형식으로 보기에 적절하다.              - 정답
    3) RTF 형식보다 호환성이 높은 텍스트 문서 형식이다.
    4) TXT를 편집할 수 있는 문서 편집기라면 조작이 가능하다.
  
  4. 스탠포드 대학의 도널드 크누스가 개발한 상세 지정이 가능한 문서 정형 또는 조판 시스템으로 유닉스상에서 개발 된 일괄 방식의 텍스트 형식은 ?
    1) PDF
    2) XHTML
    3) TeX            - 정답
    4) RTF
  
  5. EPUB(Electronic Publishing, ePublishing)에 대한 설명으로 맞는 것은 ?
    1) 폐쇄형 전자서적 표준이다.
    2) 소프트웨어를 자유롭게 갸발할 수 없다.
    3) 레지스터나 벡터 이미지를 담을 수 있는 기능이 없다.
    4) 메타데이터를 포함하고 있으며, DRM과 CSS를 지원한다.       - 정답
    
  6. 마크업 언어가 실제 표시되는 방법을 기술하는 언어는 ?
    1) DRM
    2) W3C
    3) CSS         - 정답
    4) HTML
```

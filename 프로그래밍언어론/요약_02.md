## 요약2_프로그래밍 언어의 발전 및 동작원리
```
  01. 튜링기계라는 상상속의 기계는 오늘날 컴퓨터와 흡사한 구조를 지니고 있었지만, 이것이 전자식 컴퓨터로 도입 된 것은 콜로서스와 에니악이다. 하지만 두 컴퓨터 모두 프로그래밍이 어려웠으므로
      저장 프로그램 방식의 에드박이 등장하였는데, 이때 비로소 프로그램이라는 개념이 등장하게 되었다.
  
  02. 초기 컴퓨터에는 컴퓨터 작업을 수행하기 위한 오퍼레이터가 필요했는데, 오퍼레이터의 일은 운영체제가 담당하게 되었으며, 운영체제는 일괄처리 방식, 시분할 방식을 거쳐, 오늘날 이용하고 있는 GUI 방식의 운영체제로
      발전하였다. 특히 최근에는 웹 시스템과 스마트 기기의 모바일 운영체제가 새로운 계산 플랫폼으로 각광받고 있다.
  
  03. 1950년대 초기 프로그래밍 언어 중에서 중요한 언어로는 Fortan, LISP, Algol이 있는데, 이에 더하여 1960년대 초 발표 된 Cobol까지 4개의 언어는 주요 프로그래밍 언어 개념을 제시하였다.
      Fortan은 수식 계산, 변수, 대입문, 함수, 제어문의 초기 형태를 제시하였으며, Algol은 제어 구조 개념을, LISP는 함수형 언어개념을 소개하였다.
      
  04. 1960년대 발표 된 Cobol은 레코드를 통한 자료 구조 개념을 제시하였는데, 레코드는 Algol의 제어 구조와 더불어 구조화 프로그래밍 개념을 정립하는 데 큰 축을 담당하였다.
      IBM은 PL/I이라는 방대한 언어를 만들고 이 언어를 이용하여 MULTICS라는 운영체제 개발을 추진하지만, 이 조합은 큰 인기를 얻지 못하였으며 대신 C와 Unix 조합이 발전하였다.
      1964년 발표 된 BASIC은 교육용 언어로서 큰 인기를 끌었으며 Simula는 객체지향 개념을 처음 소개한 것으로 유명하다.
  
  05. 1970년대의 프로그래밍 언어 중 가장 영향력이 높은 것은 Unix와 함께 등장한 C언어이다. 또한 Pascal은 구조화 프로그래밍의 교육용 언어로 자리잡았다. 이와 더불어 다양한 패러다임의 언어가 등장하였는데,
      인공지능 언어 Prolog가 1972년 등장하였으며 더욱 발전된 형태의 객체지향 언어인 Smalltalk도 같은 해에 발표되었다. 안전성을 높이기 위한 언어인 Ada 설계도 1970년대에 시작된다.
      함수형 프로그래밍 언어도 발전하기 시작했는데, 강력한 타입 시스텝을 갖춘 현대 함수형 언어인 ML이 1973년 발표되고, 간결한 형태의 LISP인 Scheme도 1975년 발표되었다.
  
  06. 1980년대에는 객체지향 개념이 더욱 견고해지고 다양한 언어 표준이 이루어졌다. 먼저 1984년에 방대한 LISP 변종을 통합하여 Common Lisp가 발표되었으며, Scheme은 독자적으로 계속 발전하였다.
      객체지향 언어 측면에서는 Smalltalk의 객체지향 개념이 C에 도입되어 Objective-C가 탄생하였고, Simula의 클래스 개념이 C에 도입되어 C++rk 탄생하였다. 또한 스크립트 언어도 발달하였는데,
      1980년대 개발 된 유명한 스크립트 언어로 Perl을 들 수 있다.
      
  07. 1990년대에는 Java와 JavaScript, Python을 필두로 하여 프로그래밍 언어의 대중화가 시작되었다. 사실 1980년대의 프로그래밍 언어는 사용하기 매우 어려웠는데, Java는 단순화 된 C++ 형태를
      추구하였으며 JavaScript는 Java보다 더 단순한 형태의 스크립트 언어로 고안되었다.
      빠른 프로토 타이핑을 지원하는 Python은 프로그래밍 초보자도 배우기 쉬울 정도로 간결한 언어이다. 지연 함수형 언어인 HasKell은 모나드 개념을 통해 Java와 Scala등 JVM 언어 발전에 기여하였다.
  
  08. 컴퓨터의 인출-해석-실행 주기를 프로그래밍 언어 구현에 도입한 것이 바로 인터프리터로, 인터프리터는 고급언어 프로그램으로 작성 된 문장을 하나씩 읽어서 수행한다.
      인터프리터의 실행 시 해석 부담을 줄이기 위해 고급언어 프로그램을 기계어로 미리 번역하는 프로그램이 컴파일러이다. 이 두 방식을 조합한 것이 하이브리드 구현인데 하이브리드 구현에서는 고급언어 프로그램을 
      중간 코드 형태로 컴파일 하고 이 중간 코드를 가상기계 상에서 해석한다.
  
  09. 프로그래밍 언어의 요구사항인 표현 풍부성, 유지 보수성, 실행 가능성을 효과적으로 지원하기 위해 따라야 하는 프로그래밍 언어 설계 원칙으로는 규칙성, 추상화 지원, 복잡도 제어 등을 들 수 있다.
  
  10. 프로그래밍 언어의 평가 기준 중 대표적인 것으로 작성력, 가독성, 신뢰성, 직교성 일관성, 확장성, 효율성, 유연성, 이식성 등을 들 수 있다. 이 중에서 직교성과 일관성은 언어 설계 원칙의 규칙성과 관련이 높다.
  
  11. 프로그래밍 언어의 평가 기준은 상충 될 수 있는데, 예를 들면, 효율성과 신뢰성, 가독성과 작성력, 신뢰성과 유연성은 상충되는 측면이 있다. 이럴 경우에 언어 설계자는 이 둘 사이의 적절한 지점에서 절충해야 한다.
  
  12. 프로그래밍 언어를 선택할 때는 언어 사용 커뮤니티가 활발하고 호의적인지, 언어의 특정 응용 분야가 있는지, 새로운 프로그래밍 패러다임을 지원하는지 등을 고려해야 한다.
  
```

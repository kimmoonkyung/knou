```
  1. 원소들의 중복을 허용하지 않고, 원소의 순서는 의미가 없는 컬렉션에 대한 기능을 정의하고 있는 인터페이스는 무엇인가?
    Set
      수학에서 집합을 구현하기 위한 것으로 생각하면 된다.
```

```
  2. 자바 API 문서를 조사하여 SortedSet 인터페이스를 구현한 클래스가 어떤 것이 있는지 알아보라.
    ConcurrentSkipListSet, TreeSet
      해설 SortedSet는 ‘오름차순으로 정렬된 집합’을 위한 인터페이스이다. Java API 문서에 해당하는 웹페이지는 http://docs.oracle.com/javase/8/docs/api/이다.
```

```
  3. 컬렉션 인터페이스나 클래스는 제네릭 타입으로 정의되어 있어서 그것을 사용할 때는 원소로 사용되는 객체의 타입을 함께 지정해야 한다. 이것의 장점을 무엇 인가?
      타입을 지정함으로써 컴파일 타임에 컬렉션에 추가되는 객체의 타입이 정확한지 검사할 수 있다. 따라서 실행 오류의 가능성이 줄어든다.
```

```
  4. 컬렉션 인터페이스 중에서 List를 설명하라.정답 | 원소의 중복을 허용하며 순서가 의미를 가지는 컬렉션을 위한 인터페이스이다.
    해설 이것을 구현한 클래스로 ArrayList, LinkedList, Stack 등이 있다.
```

```
  5. List에 존재하는 원소를 차례로 다루기 위한 방법으로 Iterator 인터페이스를 사용할 수 있다. 여기서 존재하는 세 가지 메소드는 무엇인가?
    boolean hasNext(), E next(), void remove()
      해설 Collection 인터페이스의 하위 클래스들은 컬렉션으로부터 Iterator 객체를 리턴하는 iterator() 메소드를 제공한다.
           Iterator 객체를 통해 위의 메소드를 사용할 수 있다.
```

```
  6. (키, 값) 형태의 원소를 저장하며, 원소들이 정렬되어 있는 컬렉션을 위한 인터페이스는 무엇인가?
    SortedMap 인터페이스 
      Map의 하위 인터페이스이며 사전이나 전화번호부를 위한 기능을 제공하기 위한 것이다.
```

```
  7. Map 인터페이스에 선언되어 있는 메소드 중 컬렉션에 원소를 추가하기 위한 것은 무엇인가?
      V put(K key, V value)
        컬렉션에서 해당 키를 가지는 원소가 없으면 추가된다. 만약 해당 키를 가지는 원소가 이미 존재하면 값을 변경한다. 
        한편 저장된 원소의 값을 읽기 위한 메소드는 ‘V get(Object key)’이다.
```

```
  8. (이름, 학번) 형태의 원소로 구성된 집합을 다루기 위해 HashMap 객체를 생성하고자 한다. 선언문을 작성하라. 단, 이름은 문자열이며 학번은 정수이다.
    Map<String, Integer> sample = new HashMap<String, Integer>();
```

```
  1. 다음 중 잘못이 있는 것을 모두 고르시오.
            a. final이면서 abstract 클래스
            b. static이면서 abstract 필드
            c. public이면서 synchronized 메소드
            d. final이면서 abstract 메소드
    a, b, d
      해설 final 클래스는 자식에게 상속될 수 없고 abstract 클래스는 상속되어 사용되어야 한다. 
           따라서 a는 모순이다. 
           abstract 필드라는 것은 존재하지 않으므로 b가 잘못되었다. 
           서브 클래스에서는 부모로부터 상속받은 final 메소드를 재정의할 수 없으나 abstract 메소드를 재정의해야 한다. 따라서 d는 모순이다.
```

```
  2.  클래스가 인터페이스를 상속받을 때 사용해야 하는 키워드는 무엇인가? 다중 상속이 가능한가?
    implements이며 다중 상속 가능
      해설 하나의 클래스 또는 하나의 인터페이스가 여러 슈퍼 인터페이스를 상속받을 수 있다. 
           즉, 인터페이스를 상속받는 경우에는 다중 상속이 가능하다. 
           한편 서브 인터페이스가 슈퍼 인터페이스를 상속받을 때는 extends 키워드를 사용한다.
```

```
  3. 추상 클래스와 인터페이스의 차이점을 설명한 것이다. 잘못된 것은?
              a. 추상 클래스는 static final이 아닌 데이터도 포함할 수 있다.                       
              b. 추상 클래스는 몸체가 구현된 메소드도 포함할 수 있다.                               -- 인터페이스는 모든 메소드가 추상 메소드다
              c. 추상 클래스는 반드시 public 클래스일 필요가 없다.                                  -- 인터페이스는 public 이어야 한다
              d. 추상 클래스에 존재하는 추상 메소드는 서브 클래스에서 구현될 필요가 없다.
    d 
      해설 인터페이스의 경우에는 public static final인 필드만 포함하며 모든 메소드가 추상 메소드이다. 
           또한 인터페이스는 public이어야 한다. 
           추상 클래스나 인터페이스를 상속받는 클래스(추상 클래스가 아닌 경우)는 상속되는 모든 추상 메소드의 몸체를 만들어 주어야 한다.
```

```
  5.  A와 B가 클래스이고 X, Y, Z가 인터페이스라고 가정할 때, 다음 중 잘못된 것을 모두 고르시오.
            a. class A extends B  {  } 
            b. class A implements X, Y  {  }
            c. interface X extends Y, Z  {  } 
            d. interface X extends A  {  }
    d
      해설 클래스가 인터페이스를 상속받을 때 그리고 인터페이스가 다른 인터페이스를 상속받을 때는 다중 상속이 가능하다. 
           인터페이스가 클래스를 상속하는 것은 의미적으로 모순되므로 그러한 경우는 없다.
```
